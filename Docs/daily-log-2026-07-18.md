## Notes

- The Xcode project uses filesystem-synchronized groups, so the new Swift files under `Fitness Buddy/Views` are picked up without manually editing `project.pbxproj`.
- Git status still reflects the earlier folder reorganization as deleted root files plus untracked app folders. That was already present before this refactor and was left untouched.
- Product decision: a typed entry can represent a whole meal or multiple ingredients. The app should not interrupt users to group every line item.
- Product decision: avoid persistent submit controls inside the notes surface while the keyboard is open; the nutrition home should feel like writing first, logging second.
- Barcode entry will need a product database lookup after scanning. Options to evaluate later: Open Food Facts for a free/open MVP, Nutritionix or Edamam for commercial coverage, or a hybrid with local user-saved foods for misses.
- Product decision: onboarding should collect goals/stats later, but Settings can own editable macro targets now so users are not blocked before onboarding is polished.
- Product decision: the character-growth concept is scrapped. Fitness Buddy is now a simple tracker app with Nutrition, Workouts, macro goals/settings, and clean onboarding later.
